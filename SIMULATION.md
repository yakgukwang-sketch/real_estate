## 2. 사람: local_agent.py

### 핵심 변경
- **구역(zone) 개념 제거** → 실제 건물 좌표로 직접 이동
- **상권정보 + 건축물대장 통합** → 599개 건물 (상업 583 + 비상업 16)
- **다익스트라 1회 사전 계산** → apt에서 모든 노드까지 최단 경로 캐시

### 목적지 선택 로직
1. 동기에 맞는 유형 건물만 후보 필터링
2. 거리 선호에 따른 가중치: near(지수감쇠), mid(가우시안), far(역지수)
3. 가게 수(인기도) 가중치: `sqrt(min(store_count, 20))`
4. 가중 확률로 랜덤 선택

## 3. 시각화: 08_local_sim.py

- Leaflet Canvas 렌더러 + `requestAnimationFrame` 기반 60fps 애니메이션
- 에이전트가 인도 네트워크 경로 위를 **끊임없이 순환 이동** (갈 때→올 때 반복)
- 거리 기반 보간으로 균일한 속도 이동
- 목적지 유형별 색상: 빨강(음식점), 파랑(학원), 주황(상점), 초록(병원), 보라(생활서비스), 회색(기타)
- 인도 네트워크 + 횡단보도(점선) 오버레이
- 사이드바: 동기별 집계, 도보시간 분포, 개별 에이전트 추적

## 4. 소비 시뮬레이션 (v2, 2026-03-20)

에이전트 이동과 소비가 통합됨. 각 이동에 `spend` 필드가 포함.

- 업종별 기본 객단가: 음식점 9,000원, 상점 15,000원, 대형상가 30,000원 등
- 프로파일별 소비 패턴 차별화 (직장인 > 주부 > 맞벌이 > 은퇴자 > 학생)
- `spending_summary()` 함수로 총소비/시간대별/프로파일별 집계
