
## 6. 현재 한계 및 개선 사항

### 해결된 문제
- ~~단일 출발점~~ → v3에서 다중 출발점 모델
- ~~외부 유입 없음~~ → 학원학생/외부직장인/방문객 프로파일 추가
- ~~시간대 객단가 동일~~ → 저녁/회식 보정 적용
- ~~5개 유형만~~ → 대치동 8개, 영등포 5개 프로파일

### 남은 과제
1. **습관/선호 모델링** — 매일 같은 카페 가는 패턴 없음
2. **날씨/요일/계절 변동** — 고정 스케줄만 사용
3. **배달/온라인 소비** — 도보 방문만 시뮬레이션
4. **건물 이름 정제** — 영등포 일부 건물명 누락 (빈 문자열)
5. **도보 경로 정확도** — 20분+ 이동이 33% (대치동), 실제로는 더 짧은 경로 존재 가능
6. **식료품점 과소** — 편의점 방문 비중이 실제보다 낮음
7. **대형상가/백화점** — 대치동에 대형상가 건물 데이터 부족
